# AGENTS.md

## Architecture
- WinForms C# desktop app, .NET Framework 4.8 (not .NET / .NET Core)
- Solution uses `.slnx` format (VS 2022+ only)
- Entrypoint: `VisualCutterForm/Program.cs` → `Application.Run(new Form1())`
- `Form1` is a `partial class`; the designer file (`Form1.Designer.cs`) is auto-generated and should not be hand-edited for layout

## Dev Environment
VS Developer Command Prompt (VS 2022 Insiders):
```
C:\Program Files\Microsoft Visual Studio\18\Insiders\Common7\Tools\VsDevCmd.bat
```
PowerShell shortcut:
```powershell
& "C:\Program Files\Microsoft Visual Studio\18\Insiders\Common7\Tools\Launch-VsDevShell.ps1"
```

## Build & Run
Run these from the VS Developer Command Prompt (or after launching `VsDevCmd.bat`):
```bash
# Build (Debug)
msbuild VisualCutterForm.slnx /p:Configuration=Debug

# Build (Release)
msbuild VisualCutterForm.slnx /p:Configuration=Release

# Run
VisualCutterForm\bin\Debug\VisualCutterForm.exe
```

## Testing
- No test project exists. If tests are added, create a separate test project targeting .NET Framework 4.8.

## Code Generation
Do not hand-edit these auto-generated files:
- `Form1.Designer.cs` — regenerated by the WinForms designer
- `Properties/Resources.Designer.cs` — regenerated from `Resources.resx`
- `Properties/Settings.Designer.cs` — regenerated from `Settings.settings`

## Dependencies
- **OpenCvSharp4.Windows** (NuGet) — OpenCV bindings for image processing
- **MvCameraControl.Net** — Hikrobot MVS SDK (external ref, installed at `C:\Program Files (x86)\MVS\`)
- **System.IO.Ports** (NuGet) — serial port communication
- Project uses `PackageReference` format (not `packages.config`)

## Constraints
- No CI/CD, linter, or formatter configured
- `.csproj` uses explicit `<Compile Include="...">` entries (old format). Every new `.cs` file must be manually added to `VisualCutterForm/VisualCutterForm.csproj` or it won't be compiled.
